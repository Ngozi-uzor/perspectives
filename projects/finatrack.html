FINATrack

A personal finance dashboard for tracking income, expenses, and spending habits. Built and deployed by a four-person team in four weeks.

Full-Stack Web App React TypeScript Supabase Netlify AIMS Cameroon · 2026
View Live Demo →

My roles: Frontend development, Supabase database integration, and deployment to Netlify. FINA stands for Felicite, Idris, Ngozi, and Adam — the four team members.

The Problem

Students and young professionals often lack a consolidated view of their finances. Existing tools are either too complex, require paid subscriptions, or rely on spreadsheets that are hard to maintain. FINATrack was built to fill that gap: a lightweight, free, accessible tool for tracking daily cash flow.

Tech Stack

Frontend React 19 + TypeScript
Build Tool Vite
Styling Tailwind CSS
Charts Recharts
Database Supabase (PostgreSQL)
Auth Custom + Supabase
Hosting Netlify
Language TypeScript

Features

Authentication Sign up and login with secure user sessions.
Dashboard Summary view of total balance, income, and expenses at a glance.
Transaction Management Add, edit, and delete income and expense entries with categories.
Reports and Charts Spending distribution visualised with Recharts bar and pie charts.
Budget Tracking Set spending limits per category and track progress against them.
Goals Create savings goals and monitor progress over time.
Transaction History Scrollable list of recent transactions with date filtering.
Dark Mode Full light and dark theme toggle, persisted per user.

What I Built

My contribution covered three areas: frontend, database, and deployment.

On the frontend, I worked on building and connecting the React components: the Dashboard summary view, the transaction forms, and the layout shell that ties the four views (Dashboard, Transactions, Reports, Settings) together. The interface uses Tailwind CSS for styling and handles theme switching between light and dark mode.

On the database side, I set up the Supabase project, defined the PostgreSQL schema for transactions and users, and implemented the REST API integration using Supabase's client. This replaced an earlier version that stored data only in the browser's localStorage, giving the app real persistence across devices and sessions.

For deployment, I configured the Netlify build pipeline, connected it to the project repository, and handled the environment variable setup so the Supabase credentials are injected at build time rather than hardcoded.
