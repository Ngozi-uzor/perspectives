Machine Translation for Lingala and Tigrinya

Fine-tuning mBART-50 on two low-resourced African language pairs using a fake language code strategy.

NLP Machine Translation mBART-50 Low-Resource Languages AIMS Cameroon · 2026

My role: Equal contributor, Group 6. This was a fully collaborative project. All six team members contributed equally to the work: motivation, data selection, model setup, training, evaluation, and analysis.

The Problem

Of the world's 7,000-plus living languages, fewer than 100 receive meaningful attention in NLP research. Africa alone has over 2,000 languages. Most are entirely absent from machine translation benchmarks.

Large multilingual models like mBART-50 have made low-resource translation more tractable, but only for the 50 languages they explicitly support. Lingala and Tigrinya are not among them. This project asked a direct question: can we get usable translation for these languages anyway, using what is already there?

The Strategy: Fake Language Codes

The MAFAND-MT project established a practical workaround. If a language has no native slot in a multilingual model, assign it the slot of a structurally similar language that the model does know. The model's encoder and decoder already understand that related language. The transfer happens by proximity.

We applied this strategy to two language pairs, which turned out to be a natural experiment. One pair was a good candidate for this approach. The other was not.

FamilyAfroasiatic

Targeten_XX (English)
Weak fit: different script and family

Both pairs were trained on 4,000 parallel sentences from a 5,000-pair dataset (500 validation, 500 test). The model was mBART-50, a 610M-parameter encoder-decoder Transformer. Training ran for 5 epochs with a batch size of 8, learning rate 3 × 10−5, fp16 mixed precision, and beam search (size 4) at inference.

Results

Language Pair BLEU ↑ ChrF ↑ TER ↓ METEOR ↑
Lingala → French 15.09 35.42 78.71 38.37
Tigrinya → English 3.52 20.46 99.11 20.25

Lingala–French reached BLEU 15.09, which is competitive with MAFAND-MT's Bantu baselines (Hausa–English: BLEU 16.0 on 3,098 pairs; we used 4,000). Short, high-frequency sentences translated well, with several exact matches at BLEU 100. The worst outputs showed hallucination: the model generated fluent French unrelated to the source entirely.

Tigrinya–English was a different story. BLEU 3.52, TER 99.11. A TER near 100 means the output requires nearly as many edits as the reference has tokens. That is near-random output. The model frequently failed to parse the Ge'ez script at all and fell back to generating plausible but disconnected English text.

Why the Gap

The contrast between the two pairs is the paper's central finding. Lingala succeeded because three conditions aligned: it shares a script with the proxy (Latin), it shares a language family with the proxy (both Bantu), and its target language (French) is heavily represented in mBART-50's pre-training. All three transfer vectors pointed in the right direction.

Tigrinya failed because none of those conditions held. Ge'ez script is nearly absent from mBART-50's vocabulary. Tigrinya is Afroasiatic; Swahili is Bantu. The proxy was wrong in every dimension that matters.

Fake code effectiveness scales with script compatibility and linguistic proximity between source and proxy. Proxy-language strategies are viable for linguistically related Bantu languages. Non-Latin scripts require byte-level modelling or dedicated vocabulary extension.
