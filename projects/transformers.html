Transformers in Neural Networks

The architecture behind ChatGPT: building and evaluating a Transformer from scratch for sentiment analysis.

Deep Learning Transformers Attention Mechanism NLP AIMS Cameroon · Dec 2025

My role: Equal contributor, Group 1. The project was a fully collaborative effort across all six team members: Adam, Ayanda, Blanche, Blessing, Eunice, and Laura.

The Problem with What Came Before

Before Transformers, the dominant sequence models were RNNs and LSTMs. They worked. But they had three structural weaknesses that limited how far they could scale.

⏳ Sequential Processing Each token had to wait for the previous one. Training was slow and could not be parallelised.
🧠 Memory Loss Information from early in a sequence was compressed and often lost by the time the model reached the end.
⛔ No Parallelisation The sequential dependency made it impossible to take advantage of modern GPU hardware efficiently.

The 2017 paper "Attention Is All You Need" (Vaswani et al.) replaced this entire paradigm. The key insight was that you do not need recurrence at all. Attention is enough.

The Core Innovation: Attention

The attention mechanism allows the model to look at every other token in the sequence simultaneously when processing any given token. It does this by computing a compatibility score between a query and a set of keys, then using those scores to weight the corresponding values.

Scaled Dot-Product Attention Attention(Q, K, V) = softmax( QKT / √dk ) × V Q: what you are searching for  ·  K: what each token advertises  ·  V: what each token contains

Multi-head attention runs this mechanism in parallel across several independent heads. Each head learns to attend to different aspects of the input, whether syntax, semantics, or long-range dependencies. The outputs are concatenated and projected back to the model dimension.

Architecture and Configuration

The full Transformer stacks encoder and decoder blocks. The encoder reads the input and builds a contextual representation. The decoder generates the output, attending both to its own previous outputs and to the encoder's representation.

For this project, we built a Transformer for a classification task: sentiment analysis on IMDB movie reviews. The architecture was trained from scratch with the following configuration.

Model Architecture

Embedding dimension64
Attention heads4
Feed-forward dimension128
Vocabulary size10,000
Max sequence length200
Dropout rate0.1

Training Setup

DatasetIMDB Reviews
Training samples25,000
Test samples25,000
Batch size64
OptimiserAdam
Epochs50

Results

84.3% Transformer Accuracy
50.1% RNN Baseline
+34pp Improvement over RNN

The Transformer reached 84.3% test accuracy on binary sentiment classification. The RNN baseline scored 50.1%, close to random. The gap is not marginal. It reflects a structural difference: the Transformer can attend directly to any word in the review regardless of position. The RNN has to carry information about early words through every subsequent hidden state, and much of it does not survive.

Training curves showed the Transformer converging cleanly across 50 epochs, with validation accuracy tracking training accuracy closely. No significant overfitting was observed, which the dropout layer helped prevent.

Attention + Parallelisation + Scale = the architecture powering ChatGPT, BERT, Google Search, GitHub Copilot, and AlphaFold.
