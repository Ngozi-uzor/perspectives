Sign Language Translation

A deep learning system that translates sign language gestures into text using hand landmark extraction and a Bidirectional LSTM.

Computer Vision Deep Learning MediaPipe Bi-LSTM AIMS Cameroon · 2026

My role: Team Lead, Group 5. I led the overall project coordination and was directly responsible for the data preprocessing and augmentation pipeline. I also contributed to the model architecture decisions, particularly the choice of Bidirectional LSTM layers and the regularisation strategy.

The Problem

Deaf and hard-of-hearing communities face a persistent communication barrier. Human interpreters exist, but they are not always available or affordable. The goal of this project was to build a scalable, automated system that translates sign language gestures into text, requiring no interpreter in the loop.

We worked with the LSA64 dataset: 64 signs from Argentinian Sign Language, of which we used 20 classes. The challenge is a sequence modelling problem. A sign is not a static hand shape. It is a movement over time, and the system has to understand that temporal structure to classify correctly.

Pipeline

Input Raw video sequences
Preprocessing 30 frames extracted, BGR→RGB
Feature Extraction MediaPipe: 21 landmarks × 2 hands × 3D
Input Tensor 126 features per frame
Classification Stacked Bi-LSTM
Output Predicted sign class

The key decision in feature extraction was to discard raw pixel data entirely. Instead of feeding image frames into the network, we used MediaPipe Hands to extract the skeletal geometry of the hands: 21 joint positions per hand, each with x, y, and z coordinates. Two hands gives 126 features per frame. This makes the model lightweight and invariant to lighting, background, and skin tone.

Data Preprocessing and Augmentation

This was my primary technical responsibility.

The dataset gave us 900 videos (45 per class across 20 classes). That is a modest size for a deep learning task. To improve generalisation, I implemented a two-step augmentation strategy that tripled the effective training set to 2,700 samples.

The first technique was jittering: adding small Gaussian noise (σ = 0.002) to the landmark coordinates. This simulates natural variation in hand tremor and imprecise gesture execution. The second was scaling: multiplying all coordinates by a uniform random factor between 0.98 and 1.02, simulating slight variation in the signer's distance from the camera.

Landmarks were normalised relative to the wrist joint, making the model robust to the position of the hand within the frame rather than its absolute screen location.

Model Architecture

The classifier is a stacked Bidirectional LSTM. The bidirectional structure is important: it lets the model attend to both past and future frames when classifying any given moment in the sequence. A unidirectional LSTM would only look back. Signs often have meaningful motion in both directions through time.

Architecture: Input (30 frames × 126 features) → Bi-LSTM (128 units, return sequences) → BatchNorm → Dropout(0.3) → Bi-LSTM (64 units) → BatchNorm → Dropout(0.3) → Dense(64, ReLU) → Softmax(20 classes).

Training used Adam (lr = 0.001), categorical crossentropy, up to 200 epochs with EarlyStopping (patience = 25) and ReduceLROnPlateau (patience = 10). The model was evaluated on a stratified 80/20 split.

Results

96% Overall Accuracy
0.97 Weighted F1-Score
0.98 Weighted Precision

The model achieved 96% accuracy on the held-out test set across 20 sign classes (540 test samples). Macro-averaged precision and recall were both 0.96 to 0.98, indicating consistent performance across all classes rather than a few dominant ones inflating the aggregate score.
